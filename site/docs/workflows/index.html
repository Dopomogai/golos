Workflows

Scenario guides for real dictation

Each workflow is a full path: what to do, what you should see, and how to recover when something fails. Your hold key may be fn (default), Right ⌥, Right ⌘, or F5 — Settings → General rebinds it live. Examples below say “hold key.”

Applies to v0.3.1 macOS 13+ · open source · unsigned beta

Honest limits. Context capture is permission-, provider-, and app-dependent (best-effort). Captures shorter than about 0.3 s are treated as accidental taps and discarded with no history line. History Retry never silently inserts — it regenerates text for you to copy. A green success flash means insertion was posted to macOS, not that the target app verified delivery. macOS can drop synthetic key/paste events in some fields.

Quick sentence via hold / release

Use this for short notes, chat replies, and anything you finish in one breath.

  1. Focus a text field where a cursor can blink (Notes, Mail, Slack, a browser form, and most standard macOS inputs).
  2. Hold your hold key (default fn). Keep it down while you speak.
  3. Speak the sentence naturally. Pause only if you mean to leave silence in the audio.
  4. Release the hold key to stop capture and start the pipeline.

Expected feedback

  • Recording: red→orange waveform (notch wings, or the corner pill if you use corner mode). Idle shows nothing — silence is normal when you are not dictating.
  • Processing: wings collapse, then a blue shimmer with a processing… label (elapsed seconds after a few seconds).
  • Success: a green ✓ inserted hill that fades (~1.2 s). If formatting fell back to raw, the label may read ✓ inserted raw.

Recovery

  • Nothing appeared and no processing flash: the capture may have been under the ~0.3 s accidental-tap threshold — hold longer and speak again. Or the hold key is stolen by macOS (System Settings → Keyboard → “Press 🌐/fn key to” → Do Nothing).
  • Processing then a warning notice: open the menu-bar chakra → Settings → History. Use Copy / Retry / Show audio as needed (see recover a failed dictation).
  • Green flash but empty field: insertion was posted, not verified. Click into the field, use History → Copy text, and paste manually. Check Accessibility for golos.app.
  • Cancel mid-recording: press Esc — audio is discarded, no transcription, no history line.

Hands-free long thought

Lock recording so you can walk around a long idea without holding the key down.

  1. Focus the target field and place the caret where the text should land.
  2. Press hold key + Space (e.g. fn+Space) to start locked recording. With Input Monitoring granted, Space is swallowed so you do not type a space into the field.
  3. Speak freely for as long as you need. The waveform stays in the recording state.
  4. End cleanly with either:
    • a single press of the hold key, or
    • hold key + Space again.
    Both stop locked mode, run STT (+ optional formatting), and insert.
  5. Or cancel with Esc while still recording — audio is discarded, nothing is inserted.

Lock mode is always available via hold key + Space. Optionally set [hotkey] toggle_combo = "double_fn" in ~/.golos/config.toml so a quick double-tap of the hold key also toggles lock — there is no Settings UI control for this; it is config-only. Hold key + Space always works as the combo path.

Expected feedback

  • Entering lock: same recording waveform as hold-to-talk, but release of the hold key does not stop capture.
  • Ending lock: same processing → success path as a normal release.
  • Cancel: brief cancelled notice; back to idle with no bubble.
  • During processing, Esc discards the result before insert if it has not started yet; a cancelled processing run may still appear in History for recovery.

Recovery

  • Space characters appear while locking: Input Monitoring is missing or the event tap failed — grant Input Monitoring to golos.app, then restart golos. See Troubleshooting.
  • Lock ends immediately: you may have pressed the hold key alone (that ends lock by design). Use hold+Space only to start, then leave the hold key alone until you mean to stop.
  • History retry is running: live lock/hold may show a busy warning. Wait for the retry to finish, or cancel other work first.

Continue an existing draft

When formatting is on and context is allowed, golos can see the focused input so new dictation continues the draft instead of starting from a blank slate.

  1. Confirm formatting is enabled (Settings → General → Format with LLM). Raw mode never sends draft text to the formatter.
  2. Confirm context toggles under Settings → Prompt: at least focused field text and, for mid-sentence joins, text before cursor (defaults are on when context is enabled).
  3. Click into the draft and place the caret where the new words should go (end of paragraph, mid-sentence, empty line for a new section).
  4. Dictate with hold/release or hands-free lock. Speak the continuation (“…and the deadline moves to Friday”), not a full rewrite, unless you want one.

Expected feedback

  • Same recording → processing → success UI as any other run.
  • With a good focused-field read, the formatter may match tone, avoid re-introducing the opening, and place the new prose as a continuation.
  • History detail for the run may include a truncated context snapshot when context was collected.

Caveats (best-effort)

  • Requires Accessibility and a field that exposes text via macOS accessibility APIs. Some web apps and custom controls return empty focused text.
  • Focused field text is capped (on the order of a few thousand characters). Very long documents only contribute a recent slice.
  • Text before the cursor is a short slice for placement, not the whole document.
  • If context gather times out (~3 s), the pipeline continues with empty context — you still get a transcript, just less draft-aware cleanup.

Recovery

  • If the insert ignores the draft tone: check Prompt toggles, Accessibility, and that the caret is in the field you think it is.
  • Use History → Copy and paste the final text where you want it if the app swallowed the synthetic insert.

Comment on visible content

Citation-style wording is a best-effort formatter feature: when surrounding on-screen text is available, comments about that text may start with a short verbatim blockquote, then your take.

  1. Open content you can read on screen (document, article, long message thread) and focus a place where your comment should be typed (reply box, notes pane, empty line).
  2. Keep Format with LLM on. In Settings → Prompt, leave visible text enabled (surrounding / reading context — not the same as the focused draft).
  3. Dictate a comment that refers to the text, e.g. “This paragraph about the API timeout is too vague — ask for the actual SLA.”
  4. Release or end lock. Review the insert: when context worked, you may see a leading > … quote drawn from the visible text, then your comment.

Expected feedback

  • Normal processing UI; no special “citation mode” indicator.
  • When visible text was readable, the formatter is instructed to quote only text that appears in that surrounding context — it should not invent quotes.
  • When visible text is empty (common in locked-down web UIs), you still get a normal comment without a forced quote.

No guarantees. Visible text is scraped best-effort from accessibility trees, normalized, and capped. Browser tabs may also contribute title/URL via providers when enabled. Never assume every app exposes reading context. Do not treat quoted lines as a legal or audit trail.

Recovery

  • No quote appeared: expected when visible text was inaccessible — paste the passage yourself or switch to an app that exposes text.
  • Wrong quote: dismiss and re-dictate with a clearer reference, or edit by hand; teaching corrections is covered next.

Teach a proper name

Fix a misspelled name once in place. Golos can surface a live cue and a History suggestion — nothing is learned until you approve.

  1. Dictate a sentence that includes the name (for example “Meet Mercey at noon”). Wait for the green success flash.
  2. Edit the inserted text by hand in the same field — change the wrong spelling to the right one (Mercey → Mercy). Short proper names are valid learning targets.
  3. Pause typing for a moment. With live cues enabled (default), the edit watcher polls the field for up to ~3 minutes and only fires after the text is stable between polls.
  4. When a cue pill appears (wrong → right ✓?), click it to keep the correction immediately. Focus is not stolen by the cue.
  5. Or open Settings → History → Suggestions and choose Add to corrections, Add to dictionary, or Dismiss.

Expected feedback

  • Cue only after a stable pause — mid-typing should not spam suggestions.
  • Accepting a cue promotes the pair into corrections (and related bookkeeping) so future runs can apply it.
  • Optional learning reviewer (Settings → Learning, off by default) may re-check a stable edit with another model; suggestions still need your click.

Recovery

  • No cue: stay in the same app a few seconds after editing; use History → Check for edits; confirm learning is enabled; confirm Accessibility can read the field.
  • App switch right after the fix: a pending local pair may flush on focus loss so quick edit-then-switch is not always lost.
  • Wrong suggestion: Dismiss permanently from Suggestions, or ignore the cue.
  • Teach without an edit: select the correct spelling → menu-bar → Add selection to dictionary.

Recover a failed dictation via History

Every durable run (success, failure, or cancelled-after-processing) can land in Settings → History. Failed stages keep enough state for recovery when audio or text was retained.

  1. Open the menu-bar chakra → Settings… (History is the first tab).
  2. Select the run (newest first). Read status/error badges and the raw / final / context detail pane.
  3. Choose an action:
    • Copy text — copies the best available text (prefers final, falls back to raw) to the pasteboard. Does not change focus or insert into an app.
    • Retry — re-runs the recoverable stage on a worker. Never auto-inserts into the frontmost app. On success the status reads along the lines of “Retry saved — ready to copy.”
    • Show audio — reveals the retained WAV in Finder when [audio] keep_recordings left a file on disk (default on).
  4. After a successful Retry, select the new attempt if needed, Copy text, click the target field, and paste.

What Retry can do

Situation Needs Result
STT failed WAV still on disk New transcript attempt (no silent insert)
Format failed / re-format Stored raw text New final text for copy
Insert failed Stored final or raw Text ready to copy; you paste deliberately
Accidental tap (< ~0.3 s) — No history line; nothing to retry — dictate again

Expected feedback

  • Live notices after a bad run often point you here (“open History…”, “couldn’t insert — open History to copy”).
  • If a live dictation is in progress, Retry may report busy and append no attempt line.
  • Original failure lines are kept; retries append linked attempts.

Recovery of recovery

  • No retained audio: STT cannot be re-run (privacy: golos does not invent audio). Re-speak, or enable keep recordings for next time.
  • No text to copy: empty STT with no recoverable audio — re-dictate.
  • Show audio disabled path: status “No retained audio” when the file is missing or retention was off.

Switch between raw, fast, and formatted

Three useful speeds — all controlled from Settings → General (Save applies live).

Mode How to enable What happens Best for
Raw Uncheck Format with LLM STT only; transcript inserts as recognized. Formatter and context sharing for formatting are skipped. Maximum speed, minimal cloud text sharing after STT
Fast Format on + check Fast mode Short single-line dictations (default ≤ 10 words) skip the LLM; literal corrections still apply locally. Longer lines still format. Quick replies with occasional polish on longer takes
Formatted Format on; Fast off (or speak longer than the fast threshold) LLM cleanup: fillers, punctuation, lists, corrections, tone/context when allowed Email, docs, structured thoughts
  1. Open Settings → General.
  2. Toggle Format with LLM and/or Fast mode.
  3. Save, then run a quick test sentence into a scratch field.

Expected feedback

  • Raw and fast paths usually feel snappier; full format adds roughly a second or more depending on network and model.
  • History rows can carry a fast flag when the short-path skipped the LLM.
  • Cloud STT still sends audio when you use an OpenRouter (or other cloud) backend — turning formatting off does not make cloud STT local.

Recovery

  • Over-aggressive cleanup: switch to raw or fast for that app, or fix by hand and teach a correction.
  • Format errors that still insert: you may see ✓ inserted raw when the pipeline fell back after a format problem.

Dictate English and Ukrainian

Golos accepts a comma-separated language list for STT. Bilingual EN/UK is a common setup; exact recognition quality depends on the selected model and backend.

  1. Open Settings → General.
  2. In Languages, enter en, uk (placeholder text matches this pattern). Leave empty for model auto-detect when the backend supports it.
  3. Save. Dictate a short English phrase, then a short Ukrainian phrase, into a test field.
  4. If one language is consistently weak, try another STT model from the curated OpenRouter list, or (Apple Silicon) an on-device model after an explicit local download — then re-test on your voice.

Expected feedback

  • No separate language indicator in the notch/pill — quality shows up in the transcript.
  • Dictionary terms still bias recognition for names and domain words across languages you use.

Recovery

  • Mixed-language garble: set explicit en, uk instead of empty auto-detect, or dictate one language per run.
  • Ukrainian punctuation or transliteration issues: keep formatting on for cleanup, teach corrections for proper nouns, see Better results.
  • Do not assume every listed model treats every language equally — compare with History retries and, if you like, the local benchmark harness on your own samples.

Corner mode on a Mac without a notch

Notch wings need a camera notch. On older displays — or any time you prefer a pill — use corner mode: a small draggable status pill (bottom-right by default).

  1. Open Settings → General.
  2. Set bubble style to corner (options: notch, corner).
  3. Save. Restart golos so the bubble style applies (style is read at bubble creation).
  4. Dictate once: the pill should appear only while recording / processing / success, then hide when idle.
  5. Optionally drag the pill to a comfortable corner; it is movable by the window background in corner style.

Expected feedback

  • Same state colors as notch mode: recording waveform, processing treatment, success confirmation, idle hidden.
  • If you leave style on notch on a non-notched Mac, golos falls back to the corner pill automatically.
  • Edit-learning cue pills remain clickable without stealing keyboard focus.

Recovery

  • Never see a bubble: idle is intentionally empty. Start a hold recording. Confirm “status words” / show-text is on if you want labels on the animation.
  • Style change did nothing: fully quit from the menu-bar icon and reopen (not only Settings Save).
  • Fullscreen apps: the panel is configured to join spaces / fullscreen auxiliary — if you still see nothing, check Troubleshooting.
