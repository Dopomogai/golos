Quality guide

Applies to v0.3.1

Better results

Golos is a two-stage pipeline: speech-to-text, then an optional formatting pass. The knobs below change accuracy, cleanup, privacy, and latency. None of them make every model “best” for every voice — measure on your own recordings.

Choose cloud vs local by goal

Settings → General picks the speech-to-text backend. Default is cloud via OpenRouter. On Apple Silicon you can also download an optional on-device Whisper model (~1.5 GB) and switch the backend to local MLX. Weights never download silently — only after you click Download local (~1.5 GB).

Goal Prefer Notes
Fast setup, no model download Cloud STT (OpenRouter) Needs an API key. Every dictation uploads audio for transcription.
Keep mic audio on-device for STT Local MLX (Apple Silicon) Requires a supported Apple Silicon build plus an explicit local download.
Intel Mac Cloud only The Intel beta build is cloud-only; local MLX is not available there.
Fully local end-to-end Local MLX + formatting off + learning reviewer off See the callout below — “formatting off” alone is not enough if STT is cloud.

Critical privacy truth

“Fully local” means: Apple Silicon local MLX for STT, Format with LLM off, and the optional learning reviewer off. Cloud STT still sends audio for every dictation even when formatting is off. Formatting off only stops the second-stage transcript/context (and optional formatter audio) from leaving the Mac.

Cloud and local quality vary by speaker, mic, language, and ambient noise. Treat default model choices as starting points, not guarantees. Latency and availability also change with your network and provider status — Golos does not promise fixed response times or permanent model catalogs.

Languages: en, uk, or empty auto-detect

In Settings → General, set Languages as a comma-separated list of ISO-style codes (for example en, uk, or en, uk). Leave the field empty for auto-detect.

  • Empty — the backend auto-detects language when it can.
  • One code (e.g. en or uk) — pin recognition when you mostly speak that language.
  • Several codes (e.g. en, uk) — bias toward those languages; behavior is model-specific.

Model-specific caveats (not universal promises)

How language lists are applied differs by family:

  • Some Deepgram-style cloud models accept a multi-language mode when more than one code is set.
  • Whisper-style cloud models typically take a single language code; multi-language lists may only become a prompt hint.
  • Local MLX uses one language when exactly one code is set; a multi-language list falls back to auto-detect.
  • Individual model ids can be stronger or weaker for Ukrainian, English, or code-switching. Do not assume every curated id supports every language equally.

If results look like the wrong language, try pinning a single code, switching model, or benchmarking on your own samples rather than trusting a universal language-support claim.

Raw vs Fast vs formatted

Stage 1 always produces a transcript. Stage 2 is optional LLM cleanup controlled by Format with LLM and Fast mode in Settings → General.

Raw Fast mode Formatted (full)
When Format with LLM off Formatting on + Fast mode; short single-line dictations (default ≤ 10 words) Formatting on; Fast mode off, or dictation is longer / multi-line
What runs STT only STT + local literal corrections; skips the LLM call STT + LLM cleanup (fillers, punctuation, lists, corrections, context)
Feel Lowest overhead after STT Short inserts stay snappy; long ones still go through the LLM Most polished prose and structure
Context / citations Not used for formatting Not used on the short/fast path May use enabled context (app, draft, visible text)
Cloud STT audio Still leaves the Mac if STT is cloud Same Same, plus transcript (and optional formatter audio) to the chat model

Fast mode’s word limit is config-backed (fast_mode_max_words, default 10). Multi-line raw transcripts always take the full formatter path when formatting is enabled. If the formatter fails or has no key, Golos inserts the raw transcript instead of blocking you.

When formatter audio helps — and what it costs

Settings → Prompt includes Also send the audio to the formatter ([formatting] send_audio, off by default).

When this is on, the original WAV rides along with the transcript so an audio-capable formatter model can repair garbled, incomplete, or obviously wrong STT before insertion. That is a second-stage recovery path — separate from cloud STT uploading audio for transcription.

  • Helpful when names, code identifiers, or noisy rooms regularly garble stage 1.
  • Tradeoff — more of the original audio leaves the Mac on every formatted dictation; cost and latency are higher than text-only formatting. Exact prices depend on your provider and model.
  • Requirement — the formatter model must accept audio input. If it cannot, the app logs a warning; leave the toggle off until you pick a capable model.

Prefer fixing systematic misspellings with dictionary terms and correction pairs first. Turn on formatter audio when residual STT damage still hurts and you accept the extra privacy/cost surface.

Dictionary terms vs correction pairs

Settings → Dictionary edits two different tools. Both save live into the running pipeline.

Dictionary terms

One term per line. Fed to STT as a biasing prompt and listed for the formatter so spelling stays stable.

Use for: product names, people, tech, brand casing you want recognized up front.

golos
Wispr Flow
PyObjC
mlx-whisper
Dopomogai

Correction pairs

wrong → right rows (stored as TSV). The full formatter applies them exactly; Fast mode applies the same pairs locally as literal, case-insensitive, word-boundary replacements.

Use for: recurring STT mistakes you have already observed.

wisper flow → Wispr Flow
teh → the
open router → OpenRouter
mac os → macOS

Rule of thumb: put the ideal spelling in the dictionary so recognition starts closer; add a correction pair only after you see a stable wrong form. Hand-edits can also surface as Suggestions (History) or live cue pills — promote only what you trust; nothing auto-learns.

Focused draft vs surrounding reading context

When formatting is enabled and context is on, Golos can send separate text roles to the formatter (each toggleable under Settings → Prompt). They are not the same thing.

Role What it is How it changes output
Focused field text Full accessible text of the input you are composing in (your draft) Guides tone, topic, and draft structure. The model should not re-dump the whole field — only the cleaned new dictation.
Text before cursor Short slice of that same field before the caret Continuation: mid-sentence inserts keep flowing; new sentences capitalize/break sensibly.
Visible text Surrounding / on-screen reading context only — never a silent copy of the focused field When you explicitly refer to something on screen, the formatter may lead with a short verbatim > quote, then your comment. It should not invent quotes from outside that context.
App providers Browser tab title/URL, VS Code workspace files, Finder selection, etc. Spoken filenames and “this page” references can resolve to real paths or URLs present in context — not invented ones.

Practical effect: dictating into a half-written email with focused draft on produces continuity; commenting on a browser article with visible text on can produce a short quote + take. If context is wrong or noisy, turn off the specific toggle (or all context) rather than disabling formatting entirely.

Context and transcripts leave the Mac only when LLM formatting is enabled. Accessibility coverage varies by app — empty visible or focused text is normal when the OS cannot expose it. Do not expect universal citations or perfect field reads in every application.

Input sensitivity and speaking habits

The Input sensitivity slider (Settings → General, range about 0.5–2.5, default 1.0) is display gain for the recording waveform, not a guaranteed mic preamp for the STT engine. Raise it if the notch/pill looks too quiet while you speak clearly; lower it if silent noise looks like shouting. If recognition is still weak, check System Settings microphone input level and which device Golos is using before chasing models.

Habits that usually help

  1. Hold, then speak. Start after recording has begun so the first word is not clipped; release after the last word finishes.
  2. One thought at a time for hard names. Short, deliberate phrases beat mumbled lists when teaching dictionary terms.
  3. Say structure out loud when you want lists. “First… second… third…” or clear parallel items give the full formatter material for real bullets or numbers.
  4. Use lock mode for long takes. fn+Space (or your hold key + Space) keeps recording hands-free; Esc cancels without inserting.
  5. Fix once, promote once. Hand-correct a persistent misspelling, accept the suggestion, and stop re-teaching the same pair.

Benchmark models on your own voice

Curated cloud model lists and the local MLX option are starting points. Never assume one model is always best — accent, bilingual mix, mic, and domain vocabulary move the ranking.

From a source checkout with your usual environment, use the built-in harness: 

# Record a sample of your voice (Enter to stop)
python -m dictate.bench record myvoice

# Edit samples/myvoice.txt so it matches what you actually said
# Then compare models (optionally restrict with --models)
python -m dictate.bench run
python -m dictate.bench run --models deepgram/nova-3,openai/whisper-1 --verbose

The table reports word error rate and latency per model on your samples. You can also copy retained dictation WAVs from ~/.golos/recordings/ into samples/ when recording retention is enabled. Cloud runs use your API key and incur normal provider usage; treat cost as variable, not a fixed price list.

Pick the model that wins on your WER and the latency you can live with, then lock it in Settings. Re-run after changing mics, languages, or dictionary density.

Recommended profiles

These are practical starting stacks for v0.3.1 — adjust after a short bench and a day of real use. Paths point at Settings unless noted.

Lowest latency

  • Format with LLM: off for pure speed, or keep it on with Fast mode if short inserts should skip the LLM.
  • STT: whichever backend/model your bench shows is quick enough on your machine and network.
  • Formatter audio: off.
  • Context: optional to turn down if you keep formatting for longer takes only.

Raw mode removes stage 2 entirely after STT. Cloud STT still uploads audio.

Best cleanup

  • Format with LLM: on; Fast mode: off.
  • Dictionary + corrections: dense with your real vocabulary and observed mistakes.
  • Context: on (focused field + before-cursor; visible text if you comment on screen content).
  • Formatter audio: on only if STT still mangles and you accept the privacy/cost tradeoff.

Choose STT by bench WER first; formatting cannot invent clarity that never entered the transcript.

Privacy-minimal

  • STT: local MLX on Apple Silicon (explicit download).
  • Format with LLM: off.
  • Learning reviewer: off (default).
  • Recordings: keep or disable under config/preferences based on whether you want local WAV history.

This is the fully local stack. Intel builds cannot use local MLX. Cloud STT is never privacy-minimal for audio.

Bilingual (e.g. English + Ukrainian)

  • Languages: en, uk if you code-switch often; or leave empty for auto-detect; pin a single code for monolingual sessions.
  • STT model: pick by bench on mixed samples of your voice — do not trust a universal ranking.
  • Dictionary: include proper names in both languages as you actually spell them.
  • Formatting: on if you want punctuation/lists; the formatter is instructed to keep the input language (not translate).

Local MLX multi-code lists fall back to auto-detect; cloud backends differ. Re-test after model changes.
