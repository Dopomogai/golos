Privacy & data

What leaves your Mac, and what stays

golos is cloud-first by default, with an explicit local STT option on Apple Silicon. Formatting, context reads, retained audio, and the optional learning reviewer are separate switches. This page maps the real data paths in v0.3.2 — not marketing claims.

Applies to v0.3.2 macOS 13+ · no built-in sync or encryption layer

Data-flow matrix

Each pipeline stage can send different material off-device. Turning formatting off does not stop cloud speech-to-text from receiving audio. There is no built-in end-to-end encryption of API traffic beyond normal HTTPS, and golos does not sync ~/.golos anywhere for you.

When a cloud path is enabled, audio and/or text go to the provider you configured (OpenRouter by default, or another OpenAI-compatible endpoint). golos does not claim that third parties retain nothing. Read and trust their own policies — for example OpenRouter’s privacy policy and the policies of any model vendor behind a route you pick.

What each pipeline stage may send off-device
Stage Default What may leave the Mac Stays local when
Cloud STT On (backend = "openrouter") Recorded dictation audio (16 kHz mono WAV), plus dictionary terms as a recognition prompt/bias. Not the full formatter context block. Also applies to OpenAI-compatible and Deepgram backends when selected. Switch STT to local MLX on Apple Silicon ([stt] backend = "mlx"). Intel builds are cloud-only for STT.
Local MLX STT Off until you choose it Nothing for transcription itself. Weights live under ~/.cache/huggingface after an explicit Download local (~1.5 GB) action — dictation never starts that download silently. Always for the STT step. Formatting and the learning reviewer are independent and can still send data if enabled.
Formatter On ([formatting] enabled = true) Raw transcript text plus the filtered context block when context is enabled: app/window identity (never process PID), optional browser tab URL, VS Code workspace file list, Finder selection, and the text roles below. If no API key is configured, formatting is skipped and raw text is used. [formatting] enabled = false (raw mode), or no key configured. Context providers are not gathered for the formatter when formatting is off.
Formatter send_audio Off (false) Same as the formatter, plus the original WAV attached as input_audio for garbled-transcript recovery on audio-capable models. Extra privacy and cost exposure. Leave [formatting] send_audio = false, or disable formatting entirely.
Learning reviewer Off (reviewer_enabled = false) Only when you turn it on: bounded excerpts of raw transcript, inserted text, and your edited field (capped; not a whole large document). If reviewer_send_audio is on (default true when the reviewer is enabled) and a retained WAV exists, that audio is base64-attached too. Suggestions are never auto-promoted — you approve them. Leave the reviewer off (shipped default), or disable reviewer_send_audio for text-only review. Offline deterministic text-diff learning still works without any reviewer network call.
i
History is local

Every completed or failed run is appended to ~/.golos/history.jsonl on disk. That file is not uploaded by golos. It can still contain transcripts and context snapshots you would not want in a public bug report.

Focused draft vs surrounding reading context

Accessibility text is split into separate roles on purpose. They are not interchangeable, and visible_text never silently reuses the focused field’s value.

Role What it is Cap Typical use
focused_field_text Full accessible value of the focused input — the draft you are composing (AXValue of the focused element). ≤ 4000 characters (last chunk if longer) Continue or edit the document you are already writing.
text_before_cursor Slice of that same focused field immediately before the caret (continuation placement). ≤ 500 characters Join mid-sentence without re-sending the whole draft.
visible_text Surrounding / on-screen reading context from another suitable text area in the focused window — not the focused field. Normalized (box-drawing stripped, space runs collapsed). Empty when inaccessible. ≤ 4000 characters (last chunk) Comment-on-what-you-see / citation-style formatting when the surrounding surface is readable via Accessibility.
  • These text roles, plus app/window and provider fields (browser tab URL, workspace file list, Finder selection), leave the Mac only when formatting is enabled and the matching [context] toggles allow them.
  • Process PID is kept for local edit-capture after app switch and is stripped before any formatter prompt.
  • Master switch: [context] enabled = false skips providers and AX text reads. Per-field toggles live under Settings → Prompt (and in config.toml).
  • Context is best-effort. Missing Accessibility, Automation consent, or app support means empty fields — not a guarantee of citations or perfect app awareness.

What lives in ~/.golos

All mutable app state is under your home directory so a bundled golos.app does not need the project tree. On first launch after the rename from “dictate”, an existing ~/.dictate set is copied once into ~/.golos (originals are left in place). There is no cloud backup or multi-device sync built into golos.

Path Contents Notes
~/.golos/config.toml Settings, model ids, feature flags, optional OpenRouter API key Created with mode 600 (user-only) because it may hold a key. Environment variable OPENROUTER_API_KEY wins over the file when set. Never paste this file into issues or chat.
~/.golos/history.jsonl Append-only dictation / recovery log Raw and final text, app identity, truncated context, status/stage, optional retained audio path. Grouped by run_id in Settings → History.
~/.golos/dictionary.txt Vocabulary terms (one per line) Biases STT and is listed for the formatter. Editable in Settings → Dictionary.
~/.golos/corrections.tsv wrong → right pairs Applied by the formatter (and local fast-mode literals). Human-approved promotions land here.
~/.golos/suggestions.jsonl Pending learned pairs From edit capture / optional reviewer. Approve or dismiss in History → Suggestions.
~/.golos/dismissed.jsonl Pairs you dismissed Keeps the same bad suggestion from returning.
~/.golos/recordings/ WAV archives when retention is on Layout YYYY-MM-DD/HHMMSS_mmm.wav. Only written when [audio] keep_recordings = true (default).
~/.golos/logs/ Private rotating application logs Local only. Menu Export Diagnostics… builds a redacted zip from these plus system/permission metadata — nothing uploads until you share the file. Export excludes API keys, audio, and transcript/prompt/context content.
~/.golos/dictate.lock Single-instance flock Released when the process exits. Not personal content.
~/.golos/prompt.md (optional) Custom formatter system prompt template Created if you edit the prompt in Settings.
~/.golos/learning_prompt.md (optional) Custom learning-reviewer prompt Only relevant if the reviewer is enabled.
!
Do not share secrets or personal audio

When asking for help, never paste API keys, full config.toml, private transcripts from history, or WAV files. Describe the symptom and the Settings toggles you changed — not the content of your dictations.

Recordings retention and History recovery

[audio] keep_recordings defaults to true. When on, each capture that is saved is written under ~/.golos/recordings/ and the history line may store that path (audio / audio_retained). When off, no archive file is written for that dictation.

  • STT retry in Settings → History needs a retained WAV still on disk. If retention was off (or the file was deleted), STT cannot be re-run from History — golos will not invent audio.
  • Format retry needs stored raw text; attaching audio for send_audio is optional and only works if the WAV is still present.
  • Default History Retry does not re-insert into the frontmost app. Use Copy text for an explicit paste. Show audio reveals the file in Finder when a path exists.
  • Esc while recording discards the capture with no history line. Esc while processing can write a cancelled recovery line without inserting.
  • Turning retention off reduces local disk footprint and removes one path for reviewer_send_audio / formatter audio attach on later runs. It also limits recovery for failed cloud STT.

Retention is config-only today (edit ~/.golos/config.toml); it is not a dedicated Settings toggle in the UI. Quit golos before hand-editing the file, or change the value carefully and restart so the process reloads config.

Backup, transfer, reset, and uninstall

Prefer copy first, then delete only explicit paths you intend to remove. Avoid broad recursive deletes against your home folder or whole volumes. The examples below use only ~/.golos.

Back up

Copy the whole data directory to a folder you control (external drive, encrypted disk image, etc.):

mkdir -p ~/Desktop/golos-backup
cp -R ~/.golos ~/Desktop/golos-backup/golos-$(date +%Y%m%d)

That archive may contain an API key and personal audio. Treat it like a password vault — do not email it or commit it to git.

Transfer to another Mac

  1. Install on the destination

    Install golos and complete first-run permissions on the new Mac.

  2. Quit both instances

    Quit golos on both machines (menu-bar icon → Quit).

  3. Copy ~/.golos

    Copy the folder from the old Mac into the new Mac’s home directory (replace or merge intentionally). A full folder copy preserves history, dictionary, corrections, and recordings.

  4. Re-check keys and models

    Verify the API key and STT backend. Local MLX weights under ~/.cache/huggingface are separate; re-download on Apple Silicon if needed. Rotate the OpenRouter (or other) key if the old machine will be wiped or shared.

There is no official account-based sync. Transfer is a manual file copy only.

Reset local state

To start clean while keeping a safety copy:

cp -R ~/.golos ~/Desktop/golos-backup-before-reset
# Then remove only what you want, for example history only:
rm ~/.golos/history.jsonl
# Or clear retained audio only:
rm -R ~/.golos/recordings

Removing config.toml drops Settings and any stored key; the next launch recreates a seed config. Removing dictionary/corrections forgets vocabulary you taught.

Uninstall

  1. Quit golos

    Use the menu-bar icon → Quit.

  2. Remove the app

    Drag /Applications/golos.app to the Trash (or delete the app bundle you installed).

  3. Optionally remove data (after a backup) 
    rm -R ~/.golos
  4. Optional model cache

    Inspect ~/.cache/huggingface before deleting anything — it may hold other models. Remove Whisper weights there only if you no longer need them.

  5. Permissions leftovers

    TCC entries for golos may remain in System Settings → Privacy & Security until you remove them manually; that is normal for macOS.

Privacy-minimal setup

“Fully local” for day-to-day dictation means: speech stays on-device, and no second-stage LLM or learning reviewer is called. That combination requires Apple Silicon for local MLX. Intel builds are cloud-only for STT.

  1. Local MLX STT

    Settings → General: set STT backend to local MLX and complete the explicit local model download (~1.5 GB).

  2. Formatting off

    Turn formatting off ([formatting] enabled = false). Insertions use the raw STT transcript — no chat formatter call and no formatter context upload.

  3. Reviewer off

    Leave the learning reviewer off (default: [learning] reviewer_enabled = false).

  4. Optional hardening

    [audio] keep_recordings = false if you do not need History STT retry or local WAV archives; [context] enabled = false if you want no AX/provider reads for the formatter path; [learning] enabled = false if you do not want edit capture writing suggestions files.

✓
Check the combination

Local MLX + formatting still on = audio stays for STT, but transcript and context can still leave for the formatter. Formatting off + cloud STT = no formatter upload, but audio still goes to the STT provider.

Permissions (and why two of them are sensitive)

golos checks permissions at startup; it does not silently grant them. You approve each one in System Settings → Privacy & Security. The bundled app is a separate TCC identity from Terminal — grant permissions to golos.app when you run the DMG build.

Permission Why golos asks Sensitivity
Microphone Capture 16 kHz mono audio while you hold the talk key. Expected for any dictation app. Audio is processed per the STT backend you chose and may be retained locally when keep_recordings is on.
Input Monitoring Global hold-key and fn+Space detection, including the event tap that can swallow Space/F5 so they do not type into the target app. Sensitive: allows observing keyboard events across apps. golos uses this for hotkeys, not for logging what you type into other apps. Only grant it to software you trust.
Accessibility Read focused window title and field text for context/learning; insert via synthetic typing or Cmd+V paste. Sensitive: can read UI content and drive input. Context text leaves the Mac only when formatting (and toggles) allow it. Without Accessibility, insertion and context degrade or fail.

Browser/Finder providers may also trigger a one-time Automation consent dialog. Denying simply disables that provider. Also set Keyboard → “Press 🌐/fn key to” → Do Nothing so macOS does not steal fn for system dictation or emoji.

Report a security issue

If you believe you found a vulnerability in golos, use GitHub private vulnerability reporting — do not file a full public exploit write-up.

Open a private security advisory

Include affected version, macOS/arch if relevant, and clear reproduction steps over the private channel only. Sanitize logs: never paste API keys, tokens, full config files, personal history lines, or private audio. Full policy: SECURITY.md.
