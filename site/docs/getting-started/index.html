Install & first run

Applies to v0.3.1

Getting started

Get golos from a DMG into Applications, open the unsigned beta safely, grant three macOS permissions, finish the seven-page wizard, and dictate your first sentence.

Beta, unsigned builds. v0.3.1 ships as unsigned DMGs. First launch is right-click → Open. This release is not Developer ID signed or notarized, and updates are manual via GitHub Releases — not automatic in-app replacement.

1. Choose Apple Silicon or Intel

golos requires macOS 13+. There are two installers — pick the one that matches your Mac. Apple menu → About This Mac: “Chip” means Apple Silicon (M1–M4); “Processor” with Intel means the Intel edition.

Edition Download Transcription
Apple Silicon golos-0.3.1-apple-silicon.dmg Cloud via OpenRouter by default. Optional on-device STT after you click Download local (~1.5 GB) — weights never start downloading on their own.
Intel golos-0.3.1-intel.dmg Cloud-only. OpenRouter transcription only. Local MLX is unavailable; Settings disables it with a clear reason.

Cloud vs optional local (Apple Silicon)

  • Cloud (OpenRouter) — works immediately after you add an API key. No multi-gigabyte model download. Audio for speech-to-text is sent to the selected cloud model.
  • Local MLX (optional) — only on Apple Silicon, only after an explicit Download local (~1.5 GB) in onboarding or Settings → General. Transcription audio can stay on device. Formatting and other cloud features are separate toggles.

Full release notes and checksums: latest release. Product download chooser: golos.dopomogai.com/#download.

2. Download and install

  1. Download the DMG for your architecture (links above).
  2. Open the DMG from Downloads (double-click the disk image).
  3. Drag golos onto Applications in the Finder window the DMG shows.
  4. Eject the disk image when you are done. Launch from Applications → golos (not from the mounted DMG).

golos is a menu-bar app (no Dock icon by default). Look for the chakra status icon in the menu bar after a successful open.

3. First open: right-click → Open

Because the beta is unsigned, a normal double-click may show a Gatekeeper warning that the app cannot be opened.

  1. In Applications, find golos.
  2. Right-click (or Control-click) the app.
  3. Choose Open.
  4. Confirm Open in the dialog if macOS asks again.
Do not expect a signed or notarized installer yet. Signing and notarization are planned distribution work, not part of this beta. Right-click → Open is the supported path for v0.3.1.

4. Grant three macOS permissions

macOS gates everything a dictation app does. Grant each permission to golos.app itself (a separate identity from Terminal if you also run from source). System Settings → Privacy & Security, or use the deep links from the onboarding wizard / menu-bar Permissions submenu.

Permission Why golos needs it If missing
Microphone Captures your voice for speech-to-text. Recording fails or returns silence.
Input Monitoring Sees the global hold key (fn by default) and runs the event tap that can swallow fn+Space so a space character is not typed into the target app. Hotkeys may not work while other apps are focused; without the event tap, Space may still type during the combo.
Accessibility Inserts text into other apps (synthetic typing / Cmd+V paste), reads window titles and permitted text context, and supports the learning loop after you edit. Transcription can still run, but paste/insert is blocked and context reads fail.

Also set the fn key (if you use fn)

System Settings → Keyboard → Press 🌐/fn key to → Do Nothing. Otherwise macOS steals fn for emoji picker, system dictation, or another action, and golos cannot use it reliably.

After granting permissions, quit golos from the menu-bar icon and reopen it so TCC grants fully apply. Use menu-bar → Permissions for live ✓/✗ status, or Test insertion to paste a probe string at the cursor.

5. The seven onboarding pages

On first run (when [app] onboarded is not set), golos opens a branded wizard: dark sidebar with page dots, content on the right, Back / Continue (last page: Finish). Reopen anytime from the menu-bar icon → Welcome / Setup….

  1. Welcome — “Talk to your Mac. It types.” Short overview: hold key → speak → release; optional AI cleanup; learning names and terms as you correct them.
  2. Permissions — “macOS needs three permissions.” Live status for Microphone, Input Monitoring, and Accessibility (polls about every 2 s). Each row has Open Settings. You can Continue with missing grants, but the app will not work fully until all three are granted.
  3. Hold-to-talk key — Choose fn (default), right_option, right_command, or f5. A test pad lights up when the key is detected. If fn is selected, the page reminds you to set “Press 🌐/fn key to” → Do Nothing. The choice saves and rebinds live.
  4. Connect OpenRouter — Paste an OpenRouter key (sk-or-…), then Save & validate, or on Apple Silicon click Download local (~1.5 GB) (disabled with a reason on Intel / unsupported machines). Continue is blocked until cloud is connected or a local model is downloaded and selected. Local download is optional and explicit — never silent.
  5. Formatting choice — Two cards: AI formatting (cleaner text, lists, context-aware polish; about 1–3 s extra) vs Raw & instant (skip the formatting pass; STT still follows your backend). Changeable later in Settings. Note: short dictations can also skip AI via Fast mode in Settings.
  6. Try it — Practice in a built-in text field while the wizard is focused. Hold your key, speak, release. Try key+Space for lock mode. Esc cancels while recording. Status labels show listening / processing / inserted.
  7. Done — “You’re set.” Reminder of your hold key. Optional Test insertion pastes ✅ golos insertion test at the cursor (checks Accessibility). Finish sets onboarded and closes the window.

6. Your first dictation

Click into any app with a text cursor (Notes is a good first target). Default hold key is fn unless you changed it in onboarding.

Hold and release

  1. Hold your hold-to-talk key — recording starts; the bubble shows a live waveform (notch wings or corner pill).
  2. Speak naturally.
  3. Release the key — state goes to processing (blue shimmer / “processing…”), then a green success hill such as “✓ inserted” if paste succeeded.

Idle means no bubble — nothing is shown until you dictate. That is intentional.

Hands-free lock: hold key + Space

  • Press fn+Space (or your hold key + Space) to start or toggle locked recording without holding the key down.
  • While locked, a single press of the hold key (or hold key + Space again) stops recording and runs transcription + insert.
  • If you are already in hold-to-talk recording, hold key + Space upgrades the session to locked mode (release no longer stops it).
  • With Input Monitoring and a working event tap, Space is swallowed so it does not type into the target field. Without the tap, Space may still appear as a character.

Cancel with Esc

  • During recording or locked: Esc discards the audio — no transcription. You may see a brief “cancelled” notice.
  • During processing: Esc requests cancel; if insertion has not started yet, the result is discarded. If insertion already began, it is too late to pull the text back.

Immediate repeat

You do not need to wait for the green success animation to finish. While the app is still in the short success state (or idle), press the hold key again to start the next recording immediately. A new take ends any live edit-watcher session from the previous insert.

Success means insertion was posted to the focused app’s input path — not a guarantee every app accepted every character. If text is missing, use menu-bar → Permissions, Test insertion, and Troubleshooting.

Where text can insert

golos targets most standard macOS text fields that accept Accessibility-driven typing or paste: Notes, many browsers, chat composers, code editors, and similar. It is not a universal guarantee for every custom control, sandboxed field, secure password field, or non-standard web widget.

If a field ignores insert: confirm Accessibility is granted to golos.app, focus a plain text area, try Test insertion, and see Troubleshooting.

Manual updates

v0.3.x uses GitHub Releases as the update channel. There is no automatic signed Sparkle-style replacement while builds remain unsigned.

  1. Open the latest release (the in-app check-for-updates action opens this page as well).
  2. Download the DMG for your architecture.
  3. Quit golos from the menu-bar icon.
  4. Drag the new golos into Applications, replacing the old app when asked.
  5. Right-click → Open if Gatekeeper prompts again after an update.

Your data lives in ~/.golos/ (config, dictionary, history, optional recordings) and is not inside the .app bundle — replacing the app does not wipe that folder.

First-five-minutes checklist

  1. Confirmed Apple Silicon vs Intel and downloaded the matching v0.3.1 DMG.
  2. Dragged golos to Applications.
  3. Opened with right-click → Open (unsigned beta).
  4. Granted Microphone, Input Monitoring, and Accessibility to golos.app.
  5. If using fn: set “Press 🌐/fn key to” → Do Nothing.
  6. Finished the seven-page wizard (or connected OpenRouter / optional local STT and picked formatting).
  7. Held the hold key in Notes, spoke a short sentence, released — text appeared (or Test insertion succeeded).
  8. Tried Esc cancel once so you know the escape hatch.
  9. Bookmarked Releases for manual updates.
